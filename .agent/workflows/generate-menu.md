# 生成月度菜单

用户说"重新生成X月菜单"或"生成下月菜单"时，按此流程执行。

## 数据源

1. **灵感菜品**：从 GitHub 拉取 `inspirations.json`（`https://api.github.com/repos/zoexudesign/meal-app/contents/inspirations.json`）
2. **不爱吃食材**：从 GitHub 拉取 `dislikes.json`（`https://api.github.com/repos/zoexudesign/meal-app/contents/dislikes.json`）
3. **现有菜单**：`e:\Personal\meal-app\data.js`

## 菜单设计原则

### 早餐（固定，不需要设计）
- **希腊酸奶 + 格兰诺拉 + 水煮蛋**
- 唯一变量：水果种类（7种轮换：草莓→蓝莓→猕猴桃→橙子→苹果→香蕉→芒果）

### 午餐烹饪方式限制
- **禁止炒菜**：午餐不得使用任何"炒"的烹饪方式（包括大火炒、快炒等）
- 原因：炒菜需要洗菜、备菜、切菜、炒制、收拾厨房，总体耗时长
- **允许的烹饪方式**：煎（pan-fry/sear）、煮（boil/simmer）、蒸（steam）、烤（bake/roast）、拌（toss/mix）、焯水（blanch）
- 晚餐不受此限制

### 午餐 + 晚餐菜系比例
- **地中海饮食** ~35%（烤鸡胸沙拉、蒜虾、鹰嘴豆沙拉、烤鱼、意面等）
- **日式饮食** ~35%（三文鱼拌饭、亲子丼、味噌汤、照烧鱼、寿喜烧、乌冬面等）
- **东南亚菜** ~10%（冬阴功、越南河粉/春卷等）
- **中餐** ~10%（白切鸡、清蒸鲈鱼、香菇鸡汤等）
- **韩餐** ~10%（拌饭、荞麦冷面、寿喜烧等）

### 食材偏好与禁忌
- **不爱吃韩式辣酱做的东西**：韩式辣酱（Gochujang）只允许用在拌饭和冷面中，其他菜品不要使用韩式辣酱作为主要调味料
  - ❌ 韩式辣酱烤鸡胸、韩式辣炒鸡胸
  - ✅ 韩式拌饭（辣酱拌饭上是标配）、荞麦冷面
  - ✅ 韩式豆腐汤（순두부찌개）— 正宗做法用**辣椒面（고추가루）热油炒底油**，不放韩式辣酱
- **必须排除** `dislikes.json` 中记录的食材
- **优先使用** `inspirations.json` 中记录的灵感菜品

### 同一天食材去重
- **同一天不要三餐都出现蛋**：早餐已有水煮蛋，午餐和晚餐中最多只有一餐可以出现蛋类（煎蛋、蛋花汤等）
- 允许例外：泰式打抛饭的煎蛋（菜系标配）、拉面上的溏心蛋（菜系标配）
- 不重复连续两天的主要蛋白质来源

### 每餐菜系统一
- **同一餐中所有菜品必须属于同一菜系**，不能跨菜系混搭
  - ❌ 沙葱炒鸡蛋（中国西北）+ 鹰嘴豆沙拉（地中海）
  - ❌ 白切鸡（中餐）+ 韩式豆腐汤（韩式）
  - ✅ 清蒸鲈鱼 + 凉拌海带丝 + 米饭（统一中餐）
  - ✅ 煎鸡腿排 + 鹰嘴豆沙拉（统一地中海）

### 午餐不吃白米饭
- 午餐主食按菜系选择合适的非白米饭碳水：
  | 菜系 | 主食 |
  |------|------|
  | 中餐 | 杂粮饭 / 杂粮馒头 / 玉米 / 红薯 |
  | 日式 | 荞麦面 / 乌冬面 |
  | 韩式 | 杂粮饭 / 荞麦冷面 |
  | 东南亚 | 米粉 / 河粉 |
  | 地中海/西式 | 法棍 / 烤土豆 / 古斯古斯 |

### 健康约束
- 血糖血脂友好：多全谷物、优质蛋白、膳食纤维、Omega-3
- 每道菜做法 ≤ 15 分钟

## 生成步骤

1. 读取 `e:\Personal\meal-app\菜单设计原则.md`（所有规则的权威来源）
2. 读取 GitHub 上的 `inspirations.json` 和 `dislikes.json`
2. 确定目标月份的所有工作日（排除双休 + 公共节假日）
3. 中国公共假日参考：元旦、春节、清明、五一、端午、中秋、国庆
4. 生成菜单数据（由于数据量大，写到临时 JS 文件再用脚本合并到 `data.js`）
5. 更新 `data.js` 保留其他月份数据
6. `git add -A` → `git commit` → `git push`
7. 提醒用户刷新 app 查看

## 注意事项

- 三月和四月菜单不要完全相同，保持变化
- 每周内尽量不重复同一道主菜
- SW缓存版本号在 `sw.js` 第一行，大版本更新时需要 bump
